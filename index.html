HTML

  • HTML is the frame of the house that is website building.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • !doctype tells the browser what document type to expect when opened.
  • HTML declarations are not case sensitive, but HTML elements ARE.
  • HTML attributes are ALWAYS defined in the beginning element.
  • An element refers to everything from opening tag to closing tag.
  • A tag only refers to whats inside the brackets.

CSS

  • CSS is the style of the website.
  • Inline CSS styles an element directly in the HTML file.
  • Internal style sheet contains rules for the webpage in the head section.
  • External CSS style sheet is the most common way to style a website. It allows devs to keep all CSS rules in a separate file.
  • * applies rule to all elements on the page.
  • A margin indications how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element

Git

  • git add -A commits all files in working branch.
  • git commit commits to repository.
  • git commit -m adds a message to the commit.
  • git pull origin main pulls the base branch into a feature branch
  • git push origin BRANCHNAME pushes all changes.
  • git status: checks what branch you are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main swithces you to the main branch.

JavaScript

  • JavaScript is interactivity for the website.
  • Const creates variables that cannot be reassigned a value.
  • JavaScript is read from top to bottom, and order counts.
  • Let reassigns the value of a variable over time.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
